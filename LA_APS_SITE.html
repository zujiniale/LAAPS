01 — VISÃO GERAL

Visão Geral

Um programa de menu interativo em Java que ensina segurança digital através de dois módulos: senhas seguras e golpes online. Mínimo, direto, sem overhead.

Requisito Como atendido Status
while obrigatório Loop principal em main() — mantém o menu ativo. Segundo uso em lerInteiro() — validação de entrada. ✓ 2 usos
switch obrigatório Roteia a opção do usuário para moduloSenhas(), moduloGolpes(), saída ou mensagem de inválido. ✓ 4 casos
if obrigatório Em senhas: 3 faixas com if / else if / else. Em golpes: bifurcação com if / else. ✓ Forma completa
Menu interativo Exibido a cada iteração do while. Usuário escolhe o módulo e retorna automaticamente. ✓
Retorno ao menu Automático — ao terminar um módulo o controle volta ao while em main(). ✓
Leitura com Scanner Scanner sc estático compartilhado entre os métodos. Toda entrada usa lerInteiro(). ✓
2 tópicos do tema Módulo 1 — Senhas Seguras · Módulo 2 — Golpes Online. ✓
Saída clara Mensagens diretas, feedback com ✓ / ⚠ / ✗. Sem erros na execução normal. ✓
Filosofia do projeto A ausência de estruturas adicionais é uma escolha deliberada — não uma limitação. Cada estrutura usada tem um motivo semântico: o while é o mecanismo natural de loop de menu, o switch é o roteador correto para múltiplos casos discretos, o if classifica valores em faixas.

03 — ESTRUTURAS

Estruturas de Controle

Todas as três estruturas obrigatórias aparecem onde fazem sentido semanticamente — não apenas para "cumprir o requisito".

while OBRIGATÓRIO
main() · lerInteiro()

Dois contextos distintos. Em main(), mantém o programa vivo enquanto o usuário não digita 0 — loop natural de menu. Em lerInteiro(), rejeita entradas não-numéricas até receber um inteiro válido — validação defensiva de entrada.

switch OBRIGATÓRIO
main()

Roteia a opção para o módulo correto com 4 casos: 1, 2, 0 e default. O switch é semanticamente superior a uma cadeia de if/else if equivalente para navegação de menu com casos discretos.

if / else if / else OBRIGATÓRIO
moduloSenhas() · moduloGolpes()

Forma completa nos dois módulos. Em senhas: três níveis de classificação por faixas numéricas sobrepostas — uso canônico de else if. Em golpes: bifurcação binária com if/else.

while (!sc.hasNextInt()) EXTRA
lerInteiro()

Validação defensiva. O método lerInteiro() é uma camada protetora entre o usuário e o switch — se alguém digitar texto no menu, o programa não quebra nem lança InputMismatchException.

05 — VALIDAÇÃO

Validação de Entrada

O método lerInteiro() é a peça mais elegante do projeto — três linhas que eliminam toda a classe de erros de entrada inválida.

GuiaSegurancaDigital.java · lerInteiro() 
static int lerInteiro() {
    while (!sc.hasNextInt()) {
        System.out.print("Digite um número válido: ");
        sc.next(); // descarta o token inválido da fila
    }
    return sc.nextInt();
}
✗ Sem validação Usuário digita "abc" → InputMismatchException → programa encerra com stack trace. Professor vê erro no terminal.
✓ Com lerInteiro() Usuário digita "abc" → programa exibe "Digite um número válido:" → espera → usuário digita 1 → executa normalmente.
Por que isso importa na apresentação Durante a demo ao professor, qualquer erro de digitação poderia quebrar o programa ao vivo. O while (!sc.hasNextInt()) protege contra isso — pode digitar letras, símbolos, Enter em branco, e o programa apenas pede a entrada novamente. Estabilidade garantida.

Como o retorno funciona O retorno ao menu não é uma opção "voltar" explícita — é automático. Quando moduloSenhas() ou moduloGolpes() terminam de executar, o controle retorna para o while em main(), que exibe o menu novamente. Isso é o comportamento correto e esperado pela rubrica: "o usuário deve conseguir navegar e retornar ao menu principal."

08 — CÓDIGO COMPLETO

Código Completo

O arquivo na íntegra — ~80 linhas. Cada estrutura obrigatória está comentada inline para fácil identificação durante a apresentação.

GuiaSegurancaDigital.java · arquivo completo · ~80 linhas 
import java.util.Scanner;

public class GuiaSegurancaDigital {

    static Scanner sc = new Scanner(System.in);

    public static void main(String[] args) {
        int opcao = -1;

        // WHILE: mantém o usuário no programa até escolher sair
        while (opcao != 0) {
            System.out.println("\n========================================");
            System.out.println("   GUIA DE SEGURANÇA DIGITAL");
            System.out.println("========================================");
            System.out.println("1. Senhas Seguras");
            System.out.println("2. Golpes Online");
            System.out.println("0. Sair");
            System.out.print("Escolha uma opção: ");

            opcao = lerInteiro();

            // SWITCH: direciona para o módulo escolhido
            switch (opcao) {
                case 1:
                    moduloSenhas();
                    break;
                case 2:
                    moduloGolpes();
                    break;
                case 0:
                    System.out.println("\nAté mais! Fique seguro online.");
                    break;
                default:
                    System.out.println("Opção inválida. Tente novamente.");
            }
        }
    }

    static void moduloSenhas() {
        System.out.println("\n--- SENHAS SEGURAS ---");
        System.out.println("Uma boa senha deve ter:");
        System.out.println("- Pelo menos 12 caracteres");
        System.out.println("- Letras maiúsculas e minúsculas");
        System.out.println("- Números e símbolos (!@#$%)");
        System.out.println("- Nunca use datas de aniversário ou nomes");
        System.out.println();
        System.out.print("Quantos caracteres tem sua senha atual? ");

        int tamanho = lerInteiro();

        // IF: avalia a força da senha em três níveis
        if (tamanho >= 12) {
            System.out.println("✓ Bom! Sua senha tem um tamanho adequado.");
        } else if (tamanho >= 8) {
            System.out.println("⚠ Atenção: sua senha é razoável, mas tente aumentar para 12+ caracteres.");
        } else {
            System.out.println("✗ Sua senha é curta e vulnerável. Troque-a agora!");
        }
    }

    static void moduloGolpes() {
        System.out.println("\n--- GOLPES ONLINE ---");
        System.out.println("Tipos de golpes mais comuns:");
        System.out.println("1. Phishing (e-mail ou link falso)");
        System.out.println("2. Golpe do falso suporte técnico");
        System.out.println("3. Golpe do Pix/pagamento urgente");
        System.out.println();
        System.out.print("Você já recebeu algum desses? (1-Sim / 2-Não): ");

        int resposta = lerInteiro();

        if (resposta == 1) {
            System.out.println("\nDicas se você recebeu um golpe:");
            System.out.println("- Não clique em links suspeitos");
            System.out.println("- Não forneça dados pessoais ou senhas");
            System.out.println("- Desconfie de urgência e pressão");
            System.out.println("- Em caso de dúvida, ligue direto para a empresa oficial");
        } else {
            System.out.println("\nÓtimo! Mas fique sempre atento.");
            System.out.println("- Verifique sempre o remetente de e-mails");
            System.out.println("- Nunca compartilhe senhas ou códigos recebidos por SMS");
        }
    }

    static int lerInteiro() {
        // WHILE extra: rejeita entradas não-numéricas sem quebrar
        while (!sc.hasNextInt()) {
            System.out.print("Digite um número válido: ");
            sc.next();
        }
        return sc.nextInt();
    }
}
Hierarquia dos métodos main() nunca acessa o Scanner diretamente — toda leitura passa por lerInteiro(). Isso centraliza a validação em um único lugar. Os módulos também seguem o mesmo padrão: chamam lerInteiro() em vez de sc.nextInt() diretamente.

10 — APRESENTAÇÃO

Roteiro de Apresentação

Cinco atos, aproximadamente 5–8 minutos. Demonstre o programa rodando — não leia o código. O professor vê e ouve as estruturas ao mesmo tempo.

01
Abertura — o que é e o que usa

"Nosso projeto é um guia interativo de segurança digital em Java. O usuário aprende sobre senhas e golpes online navegando por dois módulos. Implementamos as três estruturas obrigatórias: while, switch e if."

~1 min
02
Compile e mostre o menu — aponte o while e o switch

Execute o programa. Mostre o menu aparecendo e diga: "O while mantém o usuário aqui até ele digitar 0. O switch decide qual módulo executar baseado no número escolhido."

~1 min
03
Módulo 1 — demonstre o if em três caminhos

Entre no módulo 1. Digite 6 (feedback vermelho). Volte ao menu, entre de novo, digite 9 (amarelo). Volte e tente 15 (verde). "Aqui está o if / else if / else — três caminhos diferentes."

~2 min
04
Módulo 2 — bifurcação e retorno ao menu

Entre no módulo 2. Responda 1 (já recebeu golpe) para mostrar as dicas de reação. Volte ao menu e saia com opção 0. "Mesmo padrão — if/else bifurca o fluxo. E o while deixou o usuário navegar entre módulos e voltar."

~2 min
05
Encerramento — mencione lerInteiro() como bônus

"Adicionamos um método lerInteiro() com um segundo while que protege de entradas inválidas — se alguém digitar texto, o programa não quebra. Isso cobre todos os requisitos: while, switch, if, menu, dois tópicos, retorno ao menu."

~1 min
Dica-chave de apresentação Não leia o código linha a linha — demonstre o programa rodando e aponte as estruturas enquanto elas executam. Quando o menu aparecer na tela, diga "aqui está o while". Quando entrar no módulo, diga "aqui está o switch". Quando o feedback aparecer, diga "aqui está o if". O professor vê, ouve e conecta a estrutura ao comportamento ao mesmo tempo — é a demonstração mais eficaz possível.

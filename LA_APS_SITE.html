02 — MÓDULOS

Os 2 Módulos

Cada módulo cobre um tema de segurança digital, apresenta conteúdo educativo e interage com o usuário para gerar um feedback personalizado.

MÓDULO 01 · Tecla 1
Senhas Seguras

Apresenta as boas práticas para criação de senhas fortes. Solicita o tamanho da senha atual do usuário e classifica sua força em três níveis, com recomendações específicas para cada caso.

if / else if / else Scanner.nextInt() conteúdo educativo
MÓDULO 02 · Tecla 2
Golpes Online

Descreve os três tipos de golpe mais comuns: phishing, falso suporte técnico e golpe do Pix. Pergunta se o usuário já recebeu algum desses e entrega dicas de proteção ou prevenção conforme a resposta.

if / else Scanner.nextInt() 3 tipos de golpe
Detalhe do Módulo 1 — Classificação de senhas O if / else if / else em moduloSenhas() implementa três níveis: ≥ 12 caracteres → senha adequada; ≥ 8 e < 12 → razoável, precisa melhorar; < 8 → vulnerável, troca imediata. Três caminhos distintos, lógica clara, sem ambiguidade.
Detalhe do Módulo 2 — Feedback bifurcado O if / else em moduloGolpes() separa dois fluxos: usuário que já recebeu um golpe (dicas de resposta imediata) e usuário que nunca recebeu (dicas de prevenção). O conteúdo é diferente em cada caso — não é só uma mensagem de confirmação.
03 — ESTRUTURAS

Estruturas de Controle

As três estruturas obrigatórias da disciplina, usadas de forma orgânica — cada uma onde faz sentido semanticamente, não apenas para "cumprir o requisito".

while OBRIGATÓRIO
main() · lerInteiro()

Usado em dois contextos distintos. No main(), mantém o programa vivo enquanto o usuário não digita 0 — o loop natural de qualquer menu. Em lerInteiro(), rejeita entradas não-numéricas em loop até receber um inteiro válido.

switch OBRIGATÓRIO
main()

Roteia a opção do usuário para o módulo correto. Com 4 casos (1, 2, 0 e default), o switch é a estrutura semanticamente correta para navegação por menu — mais legível que uma cadeia de if/else if equivalente.

if / else if / else OBRIGATÓRIO
moduloSenhas() · moduloGolpes()

Forma completa nos dois módulos. Em senhas, implementa três níveis de classificação com faixas numéricas sobrepostas — use case canônico de else if. Em golpes, implementa bifurcação binária com if/else.

while (!sc.hasNextInt()) EXTRA
lerInteiro()

Validação defensiva de entrada. O método lerInteiro() age como camada de segurança entre o usuário e o switch — se alguém digitar texto no menu, o programa não quebra: pede uma entrada válida até receber.

05 — VALIDAÇÃO

Validação de Entrada

O método lerInteiro() é a peça mais elegante do projeto — três linhas que eliminam toda a classe de erros de entrada inválida.

GuiaSegurancaDigital.java · lerInteiro() 
static int lerInteiro() {
    while (!sc.hasNextInt()) {
        System.out.print("Digite um número válido: ");
        sc.next(); // descarta o token inválido
    }
    return sc.nextInt();
}
Por que isso importa Sem lerInteiro(), digitar "abc" no menu causaria um InputMismatchException e quebraria o programa. Com ele, o loop while (!sc.hasNextInt()) continua pedindo entrada até receber um inteiro — o programa nunca quebra independente do que o usuário digitar.
Sem validação Usuário digita "abc" → InputMismatchException → programa encerra com erro. Professor vê stack trace no terminal.
Com lerInteiro() Usuário digita "abc" → programa exibe "Digite um número válido:" → espera → usuário digita 1 → executa normalmente.

07 — FLUXO

Fluxo de Navegação

O programa tem uma única rota de entrada e todas as saídas voltam ao mesmo ponto. Isso é feito pelo while do main().

INÍCIO │ ▼ [MENU PRINCIPAL] ← while (opcao != 0) │ ├──── 1 ──→ [SENHAS SEGURAS] │ │ Exibe boas práticas │ │ Lê tamanho da senha │ │ if/else if/else → feedback │ └──────────────────────→ MENU │ ├──── 2 ──→ [GOLPES ONLINE] │ │ Exibe tipos de golpe │ │ Pergunta se já recebeu │ │ if/else → dicas │ └──────────────────────→ MENU │ ├──── 0 ──→ "Até mais! Fique seguro online." │ └──────────────────────→ FIM │ └── outro → "Opção inválida. Tente novamente." └──────────────────────→ MENU
Observação sobre o fluxo O retorno ao menu é automático — não é uma opção "voltar" explícita. Quando moduloSenhas() ou moduloGolpes() terminam de executar, o controle volta para o while no main(), que exibe o menu novamente. É o comportamento correto e esperado pela rubrica.

09 — APRESENTAÇÃO

Roteiro de Apresentação

Quatro atos, aproximadamente 5–8 minutos. O objetivo é mostrar o programa rodando e explicar as três estruturas obrigatórias enquanto elas aparecem na tela.

01
Abertura — o que é o projeto

"Nosso projeto é um guia interativo de segurança digital em Java. O usuário aprende sobre senhas e golpes online navegando por dois módulos com feedback personalizado. Implementamos as três estruturas obrigatórias: while, switch e if."

~1 min
02
Mostrar o menu — aponte o while e o switch

Execute o programa. Mostre o menu e diga: "O while mantém o usuário aqui até ele digitar 0. O switch decide qual módulo executar baseado no número escolhido."

~1 min
03
Módulo 1 — demonstre o if e volte ao menu

Entre no módulo 1. Digite um número de senha pequeno (ex: 6) e mostre o feedback vermelho. Depois volte e tente com 15 — mostre o feedback positivo. "Aqui está o if / else if / else — três caminhos diferentes conforme o tamanho."

~2 min
04
Módulo 2 — e encerre com o relatório

Entre no módulo 2. Responda 1 (já recebeu golpe) para mostrar as dicas de reação, depois saia com opção 0. "Mesmo estrutura — if/else bifurca o fluxo. O programa encerra limpo."

~2 min
05
Encerramento — mencione lerInteiro()

"Adicionamos um método lerInteiro() com um segundo while que protege o programa de entradas inválidas — se alguém digitar texto no menu, ele não quebra. Cobre todos os requisitos: while, switch, if, menu interativo, dois tópicos, retorno ao menu."

~1 min
Dica de apresentação Não leia o código linha a linha — demonstre o programa rodando e aponte as estruturas enquanto elas executam. Quando o menu aparecer, diga "aqui está o while". Quando entrar no módulo 1, diga "aqui está o switch". Quando o feedback aparecer, diga "aqui está o if". O professor vê e ouve ao mesmo tempo.
